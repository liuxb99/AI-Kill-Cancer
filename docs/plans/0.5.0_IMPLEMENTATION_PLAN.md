# v0.5.0 — Drug Ranking Engine Implementation Plan

## Current State
- Evidence layer has CIViC + DGIdb adapters, persistence, matching priority chain
- No drug ranking capability
- No scoring system
- No ranking result persistence

## Architecture
src/backend/ranking/
- __init__.py
- scorers.py (EvidenceScorer, ResistanceScorer, SensitivityScorer, etc.)
- penalties.py (ConflictPenalty, UncertaintyPenalty)
- engine.py (DrugRankingEngine)
- models.py (Pydantic domain models)
- repository.py (DrugRankingRunRepository)

## Scoring Factors
- exact variant match, gene-level fallback penalty
- disease match, tumor type match
- evidence type, evidence direction, clinical significance
- source quality, source count, independent source count
- publication count (PMID), guideline support, regulatory approval
- resistance evidence, contraindication, conflicting evidence
- evidence freshness

## API
- POST /ranking/variant/{variant_id}
- POST /ranking/case/{case_id}
- GET /ranking/run/{run_id}
